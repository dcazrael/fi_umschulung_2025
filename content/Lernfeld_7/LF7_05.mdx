## Arduino-Sketch, DevOps & Prototyping

---

### Arduino-Sketch: Aufbau und Struktur

Ein **Arduino-Sketch** besteht aus drei Hauptbereichen: Definitionen, `setup()`- und `loop()`-Funktion. Diese Struktur ermöglicht eine einfache und klare Programmierung von Mikrocontrollern.

#### Bereich 1: Definitionen und Deklarationen

- Einbindung von **Bibliotheken** (`#include`)
- Deklaration von **Konstanten**, **Variablen**
- Definition von **Sensoren** und **Aktoren**

```cpp
#include <DHT.h>
#define DHTPIN 2
DHT dht(DHTPIN, DHT11);
```

#### Bereich 2: setup()-Funktion

Initialisierung des Systems:

- Setzen von **Pin-Modi** (`pinMode`)
- Start der **seriellen Kommunikation** (`Serial.begin`)
- Initialisierung von Sensoren/Aktoren

```cpp
void setup() {
  Serial.begin(9600);
  dht.begin();
  pinMode(13, OUTPUT);
}
```

#### Bereich 3: loop()-Funktion

Wiederholt ausgeführter **Hauptcode**:

- **Sensorwerte** lesen
- Daten **verarbeiten**
- **Aktoren** steuern

```cpp
void loop() {
  float t = dht.readTemperature();
  Serial.println(t);
  if (t > 30) {
    digitalWrite(13, HIGH);
  } else {
    digitalWrite(13, LOW);
  }
  delay(2000);
}
```

## DevOps

### Ziele von DevOps

- Schnellere **Softwarebereitstellung**
- Höhere **Qualität**
- Verbesserte **Zusammenarbeit**
- Steigerung der **Kundenzufriedenheit**

---

### Prinzipien von DevOps

- **Automatisierung** von Builds, Tests und Deployments
- Testen in **produktionsnahen Umgebungen**
- Schnelle **Feedbackschleifen**

---

### Kultur in DevOps

- **Gemeinsame Verantwortung**
- **Offene Kommunikation**
- **Kontinuierliches Lernen**

---

### Automatisierung in DevOps

- **Continuous Integration (CI)**: Häufige Code-Zusammenführungen mit automatisierten Tests
- **Continuous Delivery/Deployment (CD)**: Automatisierte Bereitstellung bis in Produktionsumgebungen
- **Monitoring und Alerts**: Frühzeitige Fehlererkennung und schnelle Reaktion

---

### Lean-Prinzipien in DevOps

- **Verschwendung reduzieren**
- Prozesse mit **Kanban** oder **Scrum** optimieren
- Fokus auf **Wertschöpfung**

---

### Messung und Monitoring

- **Metriken** wie:
  - Fehlerraten
  - Zeit zur Wiederherstellung
  - Deployment-Frequenz
- Nutzung von **Dashboards**, **Log-Analyse** und **Alerting-Systemen**

---

### Sharing in DevOps

- **Wissensaustausch** fördern
- Nutzung gemeinsamer Tools:
  - **Wikis**
  - **Dokumentationen**
  - **Meetings**

---

### DevOps-Lebenszyklus

- **Plan**: Anforderungen erfassen, Roadmaps erstellen
- **Code**: Programmieren, Versionieren, Testen
- **Build/Test**: Automatisierte **Builds**, **Unit-Tests**
- **Release/Deploy**: Stabile Releases, **kontinuierliche Bereitstellung**
- **Operate/Monitor**: Systembetrieb, Performanceanalyse, Feedback-Schleifen

---

## Prototyping in der IT

### Zweck von Prototyping

- **Visualisierung von Ideen**
- **Risikominderung**
- Identifikation von **technischen Schwächen**
- **Frühzeitiges Feedback** einholen

---

### Arten von Prototyping

- **Technik-Prototyping**:
  - Fokus auf technische Machbarkeit
  - Vorbereitung auf **Serienproduktion**

- **Service-Prototyping**:
  - Simulation und Test von **Dienstleistungen**
  - Nutzertests und Prozessevaluation

- **Software-Prototyping**:
  - Entwicklung früher Softwareversionen
  - Iteratives Feedback aus **Nutzertests**
  - Reduzierung von Fehlentwicklungen

---

**Fazit**:  
Arduino-Sketches sind ein einfacher Einstieg in die Mikrocontroller-Programmierung, DevOps fördert effiziente und zuverlässige Softwareprozesse, und Prototyping unterstützt die risikofreie Erprobung innovativer Ideen.
