## RAID (Redundant Array of Independent Disks)

> **Definition:** Technologie zur Kombination mehrerer Festplatten für Datensicherheit, Performance und Kapazität

### Vorteile

- **Ausfallsicherheit**: Schutz vor Festplattenfehlern
- **Leistungssteigerung**: Schnellere Lese-/Schreibgeschwindigkeiten
- **Datenverfügbarkeit**: Redundante Speicherung

### RAID-Implementierungen

| Typ | Vorteile | Nachteile |
|-----|----------|-----------|
| **Hardware-RAID** | Höhere Performance, einfache Migration | Höhere Kosten, Controller-Abhängigkeit |
| **Software-RAID** | Geringere Kosten, Flexibilität | Hohe CPU-Last, geringere Performance |
| **Hybrid-RAID** | Flexibilität, moderate Kosten | Kombinierte Nachteile |

## RAID-Level

### RAID 0 - Striping

```
┌─────────┐   ┌─────────┐
│ Disk 1  │   │ Disk 2  │
│ Block A │   │ Block B │
│ Block C │   │ Block D │
└─────────┘   └─────────┘
```

- **Mindestanzahl**: 2 Festplatten
- **Blockgröße**: i.d.R. 64 kByte
- **Kapazität**: `n × Festplattengröße` (100%)

**Vorteile:**
- Maximale Geschwindigkeit
- Volle Kapazitätsnutzung

**Nachteile:**
- Keine Redundanz
- Hohe Ausfallwahrscheinlichkeit

**Beispiel:**
```
2 × 1 TB = 2 TB nutzbar
```

---

### RAID 1 - Mirroring

```
┌─────────┐   ┌─────────┐
│ Disk 1  │ → │ Disk 2  │
│ Block A │   │ Block A │
│ Block B │   │ Block B │
└─────────┘   └─────────┘
```

- **Mindestanzahl**: 2 Festplatten
- **Kapazität**: `(n / 2) × Festplattengröße` (50%)

**Vorteile:**
- Hohe Datensicherheit
- Ein Ausfall tolerierbar
- Höhere Lesegeschwindigkeit möglich

**Nachteile:**
- 50% Kapazitätsverlust
- Hohe Kosten

**Beispiel:**
```
2 × 1 TB = 1 TB nutzbar
```

---

### RAID 5 - Striping mit Parität

```
┌─────────┐   ┌─────────┐   ┌─────────┐
│ Disk 1  │   │ Disk 2  │   │ Disk 3  │
│ Block A │   │ Block B │   │ Parity  │
│ Block C │   │ Parity  │   │ Block D │
│ Parity  │   │ Block E │   │ Block F │
└─────────┘   └─────────┘   └─────────┘
```

- **Mindestanzahl**: 3 Festplatten
- **Technik**: XOR-Verknüpfung für Parität
- **Kapazität**: `(n - 1) × Festplattengröße`

**Vorteile:**
- Gute Balance Performance/Redundanz
- Optimale Speichernutzung

**Nachteile:**
- Langsamere Schreibvorgänge
- Paritätsberechnung belastet System

**Beispiel:**
```
3 × 1 TB = 2 TB nutzbar
```

---

### RAID 6 - Doppelte Parität

- **Mindestanzahl**: 4 Festplatten
- **Kapazität**: `(n - 2) × Festplattengröße`

**Vorteile:**
- Zwei Ausfälle tolerierbar
- Höhere Ausfallsicherheit

**Nachteile:**
- Geringere Schreibperformance
- Mehr Overhead

**Beispiel:**
```
4 × 1 TB = 2 TB nutzbar
```

---

### RAID 10 - Mirror + Stripe

```
┌─────────┐   ┌─────────┐
│ Disk 1  │ → │ Disk 2  │  (Mirror)
│ Block A │   │ Block A │
└─────────┘   └─────────┘
     ↓             ↓
┌─────────┐   ┌─────────┐
│ Disk 3  │ → │ Disk 4  │  (Mirror)
│ Block B │   │ Block B │
└─────────┘   └─────────┘
```

- **Mindestanzahl**: 4 Festplatten
- **Kapazität**: 50%

**Vorteile:**
- Höchste Performance mit Redundanz
- Mehrere Ausfälle möglich

**Nachteile:**
- Hohe Kosten
- 50% Kapazitätsverlust

---

### JBOD (Just a Bunch of Disks)

- Keine RAID-Konfiguration
- Festplatten einzeln nutzbar
- Keine Redundanz oder Performance-Vorteil

## Redundanzkonzepte

### Hotspare

> **Definition:** Ungenutzte Festplatte im Standby für automatischen Ersatz

**Ablauf:**
```
Festplattenausfall → Hotspare aktiviert → Rebuild startet → System wiederhergestellt
```

**Vorteile:**
- Automatischer Wechsel
- Minimale Ausfallzeit

**Nachteile:**
- Zusätzliche Hardware-Kosten
- Keine Datennutzung im Standby

---

### Hotplug

- Festplatten im laufenden Betrieb tauschbar
- Kein System-Neustart nötig

---

### Cold-Standby

> **Definition:** Reserve-Festplatte außerhalb des Systems

**Vorteile:**
- Kostengünstiger
- Flexible Nutzung

**Nachteile:**
- Manuelle Intervention nötig
- Längere Ausfallzeiten
- Datenverlust-Risiko

---

## Speicherungstechniken

### Mirroring
- Identische Kopie auf zweiter Festplatte
- Verwendung in RAID 1, RAID 10

### Striping
- Datenverteilung über mehrere Festplatten
- Verwendung in RAID 0, RAID 5, RAID 6

### Parity
- XOR-Berechnung zur Wiederherstellung
- Verwendung in RAID 5, RAID 6

## Formeln zur Kapazitätsberechnung

**Legende:**
- **NK** = Nettokapazität
- **FA** = Festplattenanzahl
- **FK** = Festplattenkapazität (kleinste Festplatte)
- **k** = Anzahl RAID-Gruppen
- **g** = Gruppengröße (FA / k)

### Basis RAID-Level

**RAID 0:**
```
NK = FA × FK
```

**RAID 1:**
```
NK = FK
```

**RAID 5:**
```
NK = (FA - 1) × FK
```

**RAID 6:**
```
NK = (FA - 2) × FK
```

### Kombinierte RAID-Level

**RAID 10 (1+0):**
```
NK = (FA / 2) × FK
```

**RAID 01 (0+1):**
```
NK = (FA / 2) × FK
```

**RAID 15 (1+5):**
```
NK = ((FA / 2) - 1) × FK
```
*Mirror von 2 RAID-5-Gruppen*

**RAID 16 (1+6):**
```
NK = ((FA / 2) - 2) × FK
```
*Mirror von 2 RAID-6-Gruppen*

**RAID 50 (5+0):**
```
NK = (FA - 2) × FK
```
*Bei k=2 Gruppen: NK = k × (g - 1) × FK*

**RAID 60 (6+0):**
```
NK = (FA - 4) × FK
```
*Bei k=2 Gruppen: NK = k × (g - 2) × FK*

> **Tipp:** Größe richtet sich immer nach der kleinsten Festplatte im Array

## Kapazitätsberechnung - Beispiele

> **Annahme:** Alle Beispiele verwenden FK = 3 TiB pro Festplatte und k = 2 RAID-Gruppen bei kombinierten Systemen

| RAID-Level | FA | FK | Raw | Formel | NK |
|------------|----|----|-----|--------|-----|
| **RAID 0** | 4 | 3 TiB | 12 TiB | `NK = FA × FK` | **12 TiB** |
| **RAID 1** | 3 | 3 TiB | 9 TiB | `NK = FK` | **3 TiB** |
| **RAID 5** | 4 | 3 TiB | 12 TiB | `NK = (FA - 1) × FK` | **9 TiB** |
| **RAID 6** | 7 | 3 TiB | 21 TiB | `NK = (FA - 2) × FK` | **15 TiB** |
| **RAID 1+0** | 6 | 3 TiB | 18 TiB | `NK = (FA / 2) × FK` | **9 TiB** |
| **RAID 0+1** | 8 | 3 TiB | 24 TiB | `NK = (FA / 2) × FK` | **12 TiB** |
| **RAID 1+5** | 6 | 3 TiB | 18 TiB | `NK = ((FA / 2) - 1) × FK` | **6 TiB** |
| **RAID 1+6** | 10 | 3 TiB | 30 TiB | `NK = ((FA / 2) - 2) × FK` | **9 TiB** |
| **RAID 5+0** | 12 | 3 TiB | 36 TiB | `NK = (FA - 2) × FK` | **30 TiB** |
| **RAID 6+0** | 8 | 3 TiB | 24 TiB | `NK = (FA - 4) × FK` | **12 TiB** |

**Erklärung kombinierte RAIDs:**
- **RAID 1+5**: Zwei RAID-5-Gruppen werden gespiegelt → `g = FA/2`, NK einer Gruppe = `(g - 1) × FK`
- **RAID 1+6**: Zwei RAID-6-Gruppen werden gespiegelt → `g = FA/2`, NK einer Gruppe = `(g - 2) × FK`
- **RAID 5+0**: Stripe über k RAID-5-Gruppen → `NK = k × (g - 1) × FK` = `(FA - 2) × FK` bei k=2
- **RAID 6+0**: Stripe über k RAID-6-Gruppen → `NK = k × (g - 2) × FK` = `(FA - 4) × FK` bei k=2

## Minimale Festplattenanzahl

| RAID-Level | Mindestanzahl | Erklärung |
|------------|---------------|-----------|
| **RAID 0** | **2** | Striping über mindestens 2 Festplatten |
| **RAID 1** | **2** | Spiegelung auf mindestens 2 Festplatten |
| **RAID 5** | **3** | Striping mit Parität über mindestens 3 Festplatten |
| **RAID 6** | **4** | Striping mit doppelter Parität über mindestens 4 Festplatten |
| **RAID 10 (1+0)** | **4** | 2 Mirror-Paare im Stripe (2×2) |
| **RAID 01 (0+1)** | **4** | 2 Stripe-Sets im Mirror (2×2) |
| **RAID 15 (1+5)** | **6** | 2 RAID-5-Gruppen à mindestens 3 Platten gespiegelt (2×3) |
| **RAID 16 (1+6)** | **8** | 2 RAID-6-Gruppen à mindestens 4 Platten gespiegelt (2×4) |
| **RAID 50 (5+0)** | **6** | Mindestens 2 RAID-5-Gruppen à 3 Platten im Stripe (2×3) |
| **RAID 60 (6+0)** | **8** | Mindestens 2 RAID-6-Gruppen à 4 Platten im Stripe (2×4) |

> **Hinweis:** Bei kombinierten RAID-Leveln werden mindestens 2 Gruppen benötigt. Weitere Festplatten können hinzugefügt werden, sollten aber die Gruppenstruktur beibehalten.

## Speichertechnologien nach RAID-Level

| RAID-Level | Striping | Mirroring | Single Parity | Double Parity |
|------------|----------|-----------|---------------|---------------|
| **RAID 0** | ✅ | ❌ | ❌ | ❌ |
| **RAID 1** | ❌ | ✅ | ❌ | ❌ |
| **RAID 5** | ✅ | ❌ | ✅ | ❌ |
| **RAID 6** | ✅ | ❌ | ❌ | ✅ |
| **RAID 10** | ✅ | ✅ | ❌ | ❌ |

**Erklärung:**
- **Striping**: Datenverteilung über mehrere Festplatten
- **Mirroring**: Datenspiegelung auf mehrere Festplatten
- **Single Parity**: Eine Paritätsinformation für Wiederherstellung
- **Double Parity**: Zwei Paritätsinformationen für höhere Sicherheit
