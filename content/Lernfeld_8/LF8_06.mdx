## Java - Geschichte und Grundlagen

---

### Entstehung von Java

- Die ersten Vorarbeiten zu **Java** begannen bereits **1990**.
- Das Projekt lief bei Sun Microsystems unter dem Code-Namen "**Oak**".
- Das ursprüngliche Ziel war eine Programmierplattform für **Play Stations und TV Settop Boxen**.
- Der Name "**Java**" (Kaffee) wurde 1992 von der Produktmanagerin Kim Polese kreiert.
- Ein lauffähiger Java-Prototyp war **1994** intern bei Sun verfügbar.
- Am **23. Mai 1995** stellten John Gage und Netscape-Gründer Marc Andreessen Java der Öffentlichkeit vor.
- Der "Siegeszug" von Java begann, weil Java-Programme auf **Unix, Mac und Windows** laufen können (dank der **JVM** - Java Virtual Machine - auf den Clients).
- **Oracle**, das Sun Microsystems im Januar 2010 kaufte, entwickelt die Java-Technik mit der Java Community in Richtung **freie und quelloffene Software (OpenJDK)** weiter.

---

### Grundgerüst eines Java-Programms

- Die **Klasse** ist die Grundeinheit der objektorientierten Programmierung. Das Schlüsselwort dafür ist `class`.
- **Geschweifte Klammern `{}`** strukturieren das Programm in Code-Blöcke.
- Der Inhalt zwischen `{` und `}` gehört zum aktuellen Bezeichner (z. B. zur Klasse).
- Der Code zwischen `{` und `}` wird per Konvention eingerückt.
- Die **`main()`-Methode** gibt es in jedem Programm genau einmal.
- **Kommentare** werden während der Programmausführung ignoriert. Sie dienen der Erklärung des Programms für die Entwickler.
- **`System.out.println()`** schreibt eine Zeile auf die Konsole. Es ist eine Methode und steht für "print line".

### Variablen

- Variablen sind **Behälter (Container) für Daten**.
- Sie benötigen einen **eindeutigen Variablennamen**.
- Das Format zur Deklaration ist: `<Datentyp> <Bezeichner>;` z. B. `String output;`.
- Die Kurzschreibweise zur Deklaration und Initialisierung ist: `<Datentyp> <Bezeichner> = <Wert>;` z. B. `String output = "Dies ist ein Volkswagen";`.
- **Deklarieren** ist das Anlegen einer Variable.
- **Initialisieren** ist die Wertzuweisung einer Variable.
- Der Datentyp wird nur einmal initial angegeben und nach der Initialisierung nicht erneut hingeschrieben.
- Variablen können jederzeit **neue Werte zugewiesen** bekommen.

---

### Datentypen

- **String**: Repräsentiert Zeichenketten (Text).
- **Ganzzahlen (integer)**: Repräsentiert ganze Zahlen.
- **Gleitkommazahlen**: Repräsentiert Zahlen mit Nachkommastellen.
  - Werden durch den Datentyp **`double`** repräsentiert.
  - Es wird ein **Punkt `.`** statt eines Kommas `,` zur Trennung von Vor- und Nachkommastellen verwendet (z. B. `1.234`).
  - Bei Gleitkommazahlen kann es zu **Rundungsungenauigkeiten** kommen.
- **Boolean**: Repräsentiert **Wahrheitswerte** ("Wahr oder Falsch").
  - Kann entweder **`true` oder `false`** sein.
  - Wird verwendet, um **Bedingungen** zu überprüfen.
  - Java interpretiert Zahlenwerte (wie 0 oder 1) **nicht als boolean**.

---

### Operationen

- **Verkettung von Strings**: Texte können aneinandergehängt werden. Auf Leerzeichen im String (Text) achten.
- **Addition von Ganzzahlen**: `+` addiert Zahlen.
- **Grundrechenarten**: Addition `+`, Subtraktion `-`, Multiplikation `*`, Division `/`.
- **Zuweisung**: Der Wert auf der rechten Seite wird der Variablen auf der linken Seite zugewiesen (`=`).
- **Kurzschreibweisen für Zuweisungen**:
  - Addition: `+=`
  - Subtraktion: `-=`
  - Multiplikation: `*=`
  - Division: `/=`
  - Inkrementierung (erhöht um 1): `++`
  - Dekrementierung (verringert um 1): `--`
- **Division mit `int`**: Berechnet nur den **ganzzahligen Anteil**.
- **Modulo (%)**: Berechnet den **Rest** bei ganzzahliger Division (z. B. 7 % 3 = 1).
- **Division mit `double`**: Berechnet das exakte Ergebnis mit Nachkommastellen.
- **Mix verschiedener Datentypen** ist erlaubt.

---

### Boolesche Logik (mit `boolean`)

- **Negation (!)**: Vertauscht den Wahrheitswert.
  - `!wahr` → `falsch`
  - `!falsch` → `wahr`
- **UND (&&)**: Beide Bedingungen müssen erfüllt sein.
  - `wahr && wahr` → `wahr`
  - `falsch && wahr` → `falsch`
  - `wahr && falsch` → `falsch`
  - `falsch && falsch` → `falsch`
- **ODER (||)**: Mindestens eine Bedingung muss erfüllt sein.
  - `wahr || wahr` → `wahr`
  - `falsch || wahr` → `wahr`
  - `wahr || falsch` → `wahr`
  - `falsch || falsch` → `falsch`
- **Gleichheit (==)** : Bedingung "gleich" muss erfüllt sein.
  - `wahr == wahr` → `wahr`
  - `falsch == wahr` → `falsch`
  - `wahr == falsch` → `falsch`
  - `falsch == falsch` → `wahr`
- **Ungleichheit (!=)**: Bedingungen müssen unterschiedlich erfüllt sein.
  - `wahr != wahr` → `falsch`
  - `falsch != wahr` → `wahr`
  - `wahr != falsch` → `wahr`
  - `falsch != falsch` → `falsch`
